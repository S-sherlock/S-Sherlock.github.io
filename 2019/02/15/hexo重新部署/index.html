hexo重新部署

发表于 2019-02-15 | 分类于 hexo

背景

2019-2-15 农历正月十一，过完农历新年，迈入新的一年。换了新电脑，重新把我的老戴尔拿出来。

重新部署个人博客。

步骤

迁移

把原电脑里的博客文件夹复制到新电脑。

环境部署

  1. 安装git，可以从git官网上下载git

  2. 打开git bash，设置用户名称和邮件地址

  3. 在用户主目录下运行：ssh-keygen -t rsa -C “youremail@example.com” 把其中的邮件地址换成自己的邮件地址，然后一路回车(这里的主目录一般是指Administrator目录)

  4. 最后完成后，会在用户主目录下生成.ssh目录，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH key密钥对，id_rsa是私钥，千万不能泄露出去，id_rsa.pub是公钥，可以放心地告诉任何人。
  5. 登陆GitHub，打开「Settings」->「SSH and GPG keys」，然后点击「new SSH key」，填上任意Title，在Key文本框里粘贴公钥id_rsa.pub文件的内容（千万不要粘贴成私钥了！），最后点击「Add SSH Key」，你就应该看到已经添加的Key。
  6. 安装node.js(从官网上下载)这里有两个版本，一般选择LTS
  7. 安装hexo：打开git bash客户端，输入 npm install hexo-cli -g，开始安装hexo

如果安装过git和生成过公钥的话直接完成5-7就行，如果安装过node.js就直接安装hexo就行

更新博客

上面都完成就应该没什么问题了，在博客文件夹目录下运行git bash，然后就可以写新博客了。

第一次更新hexo g -d的时候可能会出现一个提示框，输入yes点ok就好。
